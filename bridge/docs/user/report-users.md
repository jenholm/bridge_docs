# Report users

This page is for users who only need to open and read reports.

If your job is to log in, click a report, and review the output, this is the only page you need.

## What you will usually see

After signing in, you will normally use only two parts of the app:
- Account
- Reports

The Reports page is where your assigned reports appear.

## How to sign in

1. Open the Bridge home page.
2. Enter your email address.
3. Enter your password.
4. Click Sign in.
5. After login, open Reports from the top menu.

## How to open a report

1. Click Reports in the top navigation bar.
2. Find the row for the report you want.
3. Review the columns to confirm you have the correct item:
   - report name
   - model name
   - date created
   - date updated
4. Click View Report.

The report opens in the report viewer page.

## What happens on the report page

The report page shows the finished report inside the browser.

Typical actions on this page
- read the report on screen
- scroll through the report contents
- use the PDF button if you need a downloadable copy

## Getting a PDF copy

When available, click Get PDF.

This opens or downloads a PDF version of the report. If your browser opens the PDF in a new tab, you can then save or print it from there.

## Tips for report users

- If you do not see a report you expect, ask a dev user or administrator whether it has been assigned to you.
- If a report looks old, check the Date Updated column on the Reports page.
- If a report does not open, refresh the page once and try View Report again.
- If you forget your password, use the password reset option on the site.

## What report users do not need

You do not need to use:
- Model Maintenance
- Scheduled Runs
- database-table report building screens
- upload forms

Those areas are for dev users who create and manage reports.
